# 智慧調劑台盤點網頁維護說明

這個資料夾現在只需要維護一個主要檔案：

- `index.html`：正式給藥師使用的盤點網頁，已內建調劑台位置與藥品資料。

目前不再使用入口網站，也不需要另外維護 `cabinets.json` 或多個 `oral-1.html` 之類的頁面。

## 使用者盤點流程

1. 藥師打開 GitHub Pages 網址。
2. 在上方中央選單切換調劑台，例如 `ER 調劑台 1`、`OPDAC 調劑台 3`。
3. 點選藥櫃圖上的層架或抽屜。
4. 在橫向藥品清單中選藥品。
5. 輸入數量與效期，或點選 `找不到庫存`。
6. 盤點完成後，右下角功能鍵優先使用 `分享 CSV`，把結果傳給統整藥師。

資料會自動儲存在使用者自己的手機或電腦瀏覽器中。重新整理或暫時離開網頁後，已輸入資料通常仍會保留。

## 維護者更新位置資料

當調劑台位置或藥品料位對照 Excel 有更新時，請用以下流程產生新版網頁。

1. 用瀏覽器打開目前的 `index.html`。
2. 開啟右下角功能鍵，進入 `後台設定`。
3. 先匯入 `藥品料位對照 Excel`。
   - 檔案通常是 `藥局品項料位號.xlsx`。
   - 這一步是讓料位號可以自動對應藥品名稱。
4. 再匯入 `調劑台儲位 Excel`。
   - 例如 `智慧調劑台儲位ER0427.xlsx`、`OPDAC`、`OPDB`、`UD`。
   - 可以依序匯入多個區域檔案。
   - 匯入後會自動建立該區域的多台調劑台。
5. 確認上方中央選單可以切換到各台調劑台。
6. 按 `下載已綁定盤點網頁`。
7. 將下載的新 HTML 檔重新命名為 `index.html`。
8. 上傳到 GitHub repository，覆蓋原本的 `index.html`。

完成後，所有藥師打開同一個 GitHub Pages 網址，就會看到新版位置設定。

## GitHub 覆蓋方式

最快方式：

1. 進入 GitHub repository。
2. 點開 `index.html`。
3. 點右上角鉛筆圖示或 `Edit this file`。
4. 全選舊內容並刪除。
5. 貼上新版 `index.html` 的全部內容。
6. 按 `Commit changes`。

也可以直接用 `Add file` / `Upload files` 上傳新版 `index.html`，檔名相同時會覆蓋。

GitHub Pages 有時會有短暫快取，更新後可等 1 到 3 分鐘再重新整理頁面。

## 匯出與分享 CSV

建議優先使用 `分享 CSV`：

- iOS 通常可直接分享到 LINE、Teams、Mail、雲端硬碟等 App。
- Android 若分享功能受瀏覽器限制，可改用 `匯出 CSV`，下載後再從檔案管理器傳送。

CSV 主要提供統整用，內容包含調劑台、位置、藥品名稱、料位號、數量、效期與盤點狀態。

## 注意事項

- 每位藥師的盤點輸入結果只存在自己的裝置，不會自動回傳到 GitHub。
- 維護者更新 `index.html` 不會拿到藥師手機裡已輸入的盤點資料。
- 如果藥師已經開始盤點，請避免中途要求他們重新整理或切換新版網頁。
- 如果匯入 Excel 時出現「Excel 匯入元件沒有載入」，通常是網路擋住外部套件，請換可連外網的環境產生新版 `index.html`。
